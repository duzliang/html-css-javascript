盒模型 核心 必考

标准盒模型 vs 怪异盒模型

属性 content-box (标准) border-box (IE)
width 包含 仅 content 宽度 content + padding + border
实际宽度 width + padding + border + margin width + margin
计算公式 width = content width = content + padding + border 
/* 全局重置推荐 */
*, *::before, *::after {
  box-sizing: border-box;
}

/* 组件级切换 */
.component {
  box-sizing: content-box; /* 需要精确控制内容尺寸时 */
}

盒模型组成

Content

内容区域，显示文本和图像

Padding

内边距，背景色会延伸到 padding

Border

边框，位于 padding 外，默认不占用空间

Margin

外边距，透明，不参与背景色

margin 塌陷与合并

Q: 什么是 margin 塌陷？如何解决？

塌陷场景：

  1. 父子塌陷：父元素无 padding/border 时，子元素的 margin-top 会「溢出」到父元素外
  2. 兄弟合并：相邻兄弟元素的 margin-bottom 和 margin-top 会合并取最大值
  3. 空元素：空元素的 margin-top 和 margin-bottom 会合并

解决父子塌陷：

  • 父元素添加 padding-top 或 border-top
  • 父元素 overflow: hidden
  • 父元素触发 BFC（display: flow-root）

负 margin 应用

/* 1. 元素向左/向上移动 */
.element { margin-left: -20px; }

/* 2. 扩大元素点击区域 */
.button {
  padding: 8px 16px;
  margin: -8px; /* 补偿 padding */
}

/* 3. 等高布局（圣杯布局核心） */
.column { margin-bottom: -99999px; padding-bottom: 99999px; }

/* 4. 去除列表首尾边距 */
.list-item:first-child { margin-left: -10px; }

优先级与层叠 核心 必考

优先级计算 (a, b, c, d)

等级 选择器类型 权重值
a 内联样式（style 属性） 1,0,0,0
b ID 选择器 0,1,0,0
c 类/属性/伪类选择器 0,0,1,0
d 元素/伪元素选择器 0,0,0,1
* 通配符 / 继承 0,0,0,0
!important 最高优先级（慎用） 覆盖所有
Q: 以下选择器的优先级顺序？
/* 计算权重 */
#nav .menu li a { }       /* 0,1,1,2 → 0112 */
#nav .menu .item { }      /* 0,1,2,0 → 0120 胜出 */
.nav .menu .item { }      /* 0,0,2,1 → 0021 */
#nav li a:hover { }       /* 0,1,1,2 → 0112 */
li a:hover::before { }    /* 0,0,1,3 → 0013 */

比较规则：从左到右逐位比较，高位胜出即停止。

!important 规则

  • 覆盖任何优先级，包括内联样式
  • 多个 !important 冲突时，比较选择器优先级
  • 用户代理样式表的 !important 低于作者样式
  • 用户样式表的 !important 高于作者 !important（尊重用户偏好）
  • 避免滥用：只在覆盖第三方库、工具类（utility）时使用

层叠顺序（Cascading）

当优先级相同时，后定义的样式生效：

  1. 来源：作者样式 > 用户样式 > 浏览器默认
  2. 位置：内联 > 内部样式表 > 外部样式表
  3. 加载顺序：后加载的样式覆盖先加载的（同优先级）

vmin/vmax vw 和 vh 的较小/较大值 保持比例的响应式元素
% 父元素对应属性 width/height 常用

单位 含义 支持度

rlh 根元素行高 现代浏览器

dvw/dvh 动态视口宽高（包含工具栏变化） 移动端支持
svw/svh 小视口（不考虑动态工具栏） 移动端支持
lvw/lvh 大视口（考虑工具栏收起） 移动端支持

rem 与 vw 的响应式字体方案

/* 基础设置 */
html {
  font-size: 16px;
}

/* 流体排版：在 375px-1920px 之间平滑过渡 */
@media screen and (min-width: 375px) {
  html {
    font-size: calc(16px + (24 - 16) * ((100vw - 375px) / (1920 - 375)));
  }
}

/* 限制最大最小值 */
@media screen and (min-width: 1920px) {
  html { font-size: 24px; }
}

/* Clamp 简化写法（现代浏览器） */
html {
  font-size: clamp(16px, 1vw + 0.5rem, 24px);
}

Flexbox 弹性布局 核心 必考

Flex 容器属性

属性 值 说明
display flex / inline-flex 开启 flex 布局
flex-direction row / row-reverse / column / column-reverse 主轴方向
flex-wrap nowrap / wrap / wrap-reverse 是否换行
justify-content flex-start / center / flex-end / space-between / space-around / space-evenly 主轴对齐
align-items stretch / flex-start / center / flex-end / baseline 交叉轴单行对齐
align-content stretch / flex-start / center / flex-end / space-between / space-around 多行时交叉轴对齐（需有换行）
gap <length> 项目间距（推荐，替代 margin）

Flex 项目属性

属性 作用 默认值
flex-grow 放大比例（剩余空间分配） 0（不放大）
flex-shrink 缩小比例（空间不足时） 1（允许缩小）
flex-basis 基础尺寸（定义前占据空间） auto
flex 简写：grow shrink basis 0 1 auto
align-self 单个项目对齐方式 auto
order 排列顺序（整数，默认0） 0

flex 常见简写

flex: 0 1 auto     /* 默认值，不放大可缩小 */
flex: 1            /* 1 1 0%，等分剩余空间 */
flex: auto         /* 1 1 auto，基于内容等分 */
flex: none         /* 0 0 auto，固定大小不伸缩 */
flex: 0 0 200px    /* 固定宽度 200px */
Q: flex: 1 和 flex: auto 的区别？

两者都会放大，但基础尺寸不同：

  • flex: 1 = flex: 1 1 0% - basis 为 0，忽略内容大小，严格按 grow 比例分配
  • flex: auto = flex: 1 1 auto - basis 为 auto，先按内容大小分配，剩余空间再按 grow 分配

结果差异：内容不等时，flex: 1 会使所有项目等宽；flex: auto 会先尊重内容宽度。

常见 Flex 布局模式

/* 水平垂直居中 */
.center {
  display: flex;
  justify-content: center;
  align-items: center;
}

/* 两端对齐 */
.space-between {
  display: flex;
  justify-content: space-between;
  align-items: center;
}

/* 底部固定栏 */
.sticky-footer {
  display: flex;
  flex-direction: column;
  min-height: 100vh;
}
.sticky-footer main {
  flex: 1; /* 内容区自动填充 */
}

/* 等高多列 */
.equal-height {
  display: flex;
  align-items: stretch; /* 默认，可省略 */
}

Grid 网格布局 现代 热门

Grid 与 Flexbox 对比

特性 Flexbox Grid
维度 一维（行或列） 二维（行和列同时）
内容驱动 是（基于内容尺寸） 容器驱动（先定义轨道）
项目重叠 困难 简单（显式定位）
间隙控制 gap（较新支持） gap/gap-row/gap-column
适用场景 导航、按钮组、卡片列表 页面布局、复杂网格

Grid 容器属性

/* 定义网格 */
display: grid;
grid-template-columns: 200px 1fr 2fr;  /* 三列：固定、等分、2倍 */
grid-template-columns: repeat(3, 1fr); /* 三等分列 */
grid-template-columns: repeat(auto-fill, minmax(200px, 1fr)); /* 响应式列 */

/* 行定义 */
grid-template-rows: auto 1fr auto;     /* 头、内容、尾 */

/* 简写 */
grid-template: auto 1fr auto / repeat(3, 1fr);

/* 间距 */
gap: 20px;                             /* 行列相同 */
row-gap: 20px;
column-gap: 10px;

/* 对齐 */
justify-items: start | center | end | stretch;  /* 单元格内水平对齐 */
align-items: start | center | end | stretch;    /* 单元格内垂直对齐 */
place-items: center;                            /* 简写 */

justify-content: center;                 /* 整个网格水平对齐 */
align-content: center;                   /* 整个网格垂直对齐 */

Grid 项目属性

/* 定位 */
grid-column: 1 / 3;          /* 从第1列开始，跨到第3列（1-3列） */
grid-column: 1 / span 2;     /* 从第1列开始，跨2列 */
grid-row: 1 / 2;

/* 区域命名 */
grid-area: header;           /* 使用 grid-template-areas 命名 */

/* 简写定位 */
grid-area: 1 / 2 / 3 / 4;    /* row-start / col-start / row-end / col-end */

常用 Grid 布局

/* 经典页面布局 */
.layout {
  display: grid;
  grid-template-columns: 200px 1fr 200px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header header header"
    "aside  main   sidebar"
    "footer footer footer";
  min-height: 100vh;
}

.layout header { grid-area: header; }
.layout aside { grid-area: aside; }
.layout main { grid-area: main; }
.layout .sidebar { grid-area: sidebar; }
.layout footer { grid-area: footer; }

/* 响应式卡片网格 */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(280px, 1fr));
  gap: 20px;
}
Q: Grid 的 minmax() 和 auto-fit/auto-fill 区别？
  • minmax(min, max)：定义轨道尺寸范围，可伸缩
  • auto-fill：尽可能多创建轨道，即使为空也保留
  • auto-fit：创建足够轨道后，拉伸填满容器，空轨道坍缩为 0

场景：auto-fill 适合需要保持固定列数的场景；auto-fit 适合响应式，自动调整列数以填满空间。

定位系统 核心

position 属性对比

值 脱离文档流 定位基准 堆叠上下文
static 否 无（默认值） 否
relative 否（占位保留） 自身原位置 z-index 有效时
absolute 是 最近非 static 祖先 z-index 有效时
fixed 是 视口（viewport） 是
sticky 否（滚动前） 滚动容器 + 阈值 是

sticky 定位详解

/* 顶部吸顶 */
.header {
  position: sticky;
  top: 0;
  z-index: 100;
}

/* 底部吸底 */
.footer {
  position: sticky;
  bottom: 0;
}

/* 多方向（当空间不足时） */
.element {
  position: sticky;
  top: 10px;
  bottom: 10px;
}
sticky 生效条件：
  • 祖先元素不能设置 overflow: hidden/scroll/auto（除非 overflow 在 body/html）
  • 必须指定 top/right/bottom/left 之一（不能是 auto）
  • 父元素高度要大于 sticky 元素（有滚动空间）
  • sticky 元素在文档流中（非 absolute/fixed）

z-index 与层叠上下文

/* 创建层叠上下文的方式 */
/* 1. position 非 static + z-index */
.context { position: relative; z-index: 1; }

/* 2. opacity < 1 */
.context { opacity: 0.99; }

/* 3. transform/filter/will-change */
.context { transform: translateX(0); }
.context { filter: blur(0); }
.context { will-change: transform; }

/* 4. flex/grid 子元素 + z-index */
.flex-item { z-index: 1; } /* 父是 flex 容器即可，无需 position */

/* 5. isolation */
.context { isolation: isolate; } /* 推荐，无副作用 */
Q: z-index 很大的元素被盖住的原因？

z-index 只在同一层叠上下文中比较。如果父元素创建了新的层叠上下文，子元素的 z-index 只在父上下文内有效。

层叠顺序规则（从底到顶）：

  1. 根元素背景/边框
  2. 普通流非定位元素
  3. 浮动元素
  4. 普通流定位元素（z-index: auto）
  5. 定位元素（z-index >= 0）

z-index 为负值的定位元素位于普通流元素之下。

BFC / IFC / FFC / GFC 进阶

格式化上下文（Formatting Context）

类型 触发条件 特性
BFC
块级格式化上下文		 html, float, position: absolute/fixed, display: flow-root/table-cell/inline-block, overflow 非 visible, contain: layout 内部块级盒独立布局，不受外部影响；阻止 margin 塌陷；清除浮动
IFC
行内格式化上下文		 块级容器内的行内元素自动形成 行内元素水平排列，垂直对齐方式由 vertical-align 控制，行盒高度由 line-height 决定
FFC
Flex 格式化上下文		 display: flex/inline-flex 子元素成为 flex item，一维布局，可用 flex 属性控制
GFC
Grid 格式化上下文		 display: grid/inline-grid 子元素成为 grid item，二维布局，可用 grid 属性控制

BFC 应用场景

/* 1. 清除浮动 */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}
/* 或使用 */
.container {
  display: flow-root; /* 现代方案，无副作用 */
}

/* 2. 阻止 margin 塌陷 */
.parent {
  display: flow-root; /* 或 overflow: hidden */
}
.child {
  margin-top: 20px; /* 不会塌陷到父元素外 */
}

/* 3. 自适应两栏布局 */
.sidebar {
  float: left;
  width: 200px;
}
.main {
  display: flow-root; /* 形成 BFC，不环绕浮动元素 */
}

IFC 深入理解

Q: vertical-align 为什么不生效？

vertical-align 只在 IFC（行内格式化上下文）中生效，且只对行内级元素有效。

  • 取值：baseline（默认）、top、middle、bottom、text-top、text-bottom、super、sub、百分比、长度
  • baseline：元素的基线与父元素基线对齐（图片底部空隙的原因）
  • 图片底部空隙：图片默认基线对齐，与行内 x 字符基线产生空隙。解决：vertical-align: middle/top/bottom 或 display: block

容器查询 (Container Query) 2023+

为什么需要容器查询

媒体查询基于视口尺寸，但组件可能在不同上下文中复用（侧边栏 vs 主内容区）。容器查询让组件根据自身容器的尺寸响应，实现真正的模块化响应式。

基础用法

/* 1. 定义容器 */
.container {
  container-type: inline-size; /* 或 size（宽高） */
  container-name: sidebar;     /* 可选，命名 */
}

/* 2. 容器查询 */
@container (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;
  }
}

/* 命名容器查询 */
@container sidebar (min-width: 300px) {
  .widget { /* 只在 sidebar 容器中生效 */ }
}

单位 含义
cqw 容器宽度的 1%
cqh 容器高度的 1%
cqi 容器内联尺寸的 1%（考虑书写模式）
cqb 容器块级尺寸的 1%
cqmin / cqmax cqi 和 cqb 的较小/较大值

Transform 变换 核心

2D 变换函数

函数 作用
translate(x, y) 位移，百分比基于自身尺寸
scale(x, y) 缩放，负值可翻转
rotate(angle) 旋转，deg/rad/turn
skew(x-angle, y-angle) 倾斜
matrix(a, b, c, d, e, f) 2D 变换矩阵

3D 变换

/* 开启 3D 空间 */
.container {
  perspective: 1000px;       /* 透视距离 */
  perspective-origin: center; /* 透视原点 */
}

.item {
  transform-style: preserve-3d; /* 子元素保持 3D */
  transform:
    translate3d(x, y, z)
    scale3d(x, y, z)
    rotate3d(x, y, z, angle)
    rotateX/Y/Z(angle);
}

/* 背面可见性 */
.backface-hidden {
  backface-visibility: hidden;
}

Transform 注意事项

  • 性能：transform 会触发 GPU 加速，不引起重排（reflow）
  • 层级：transform 非 none 会创建层叠上下文
  • 定位：fixed 元素在 transform 容器内会相对于该容器定位（不是 viewport）
  • 顺序：transform 函数从右向左执行，顺序影响结果

高性能动画属性

✅ 推荐（GPU 加速）

transform, opacity
只触发 Composite，不触发 Layout/Paint

⚠️ 谨慎使用

width, height, top, left, margin
触发 Layout 重排，性能开销大

✅ 较安全

color, background-color
触发 Paint，但不重排

will-change 优化

/* 提前告知浏览器准备动画 */
.animated-element {
  will-change: transform, opacity;
}

/* 动画结束后移除（避免内存占用） */
.animated-element.animation-complete {
  will-change: auto;
}
will-change 注意事项：
  • 不要过度使用，每个独立层占用内存
  • 不要应用到太多元素
  • 动画前添加，动画后移除
  • 对于即将发生的变换，提前 200-500ms 添加

减少重排（Reflow）

el.style.width = '100px';      // 读/写交错，强制同步布局
const width = el.offsetWidth;  // 强制计算
el.style.height = '200px';     // 再次重排

/* ✅ 读写分离，批量修改 */
const el = document.getElementById('app');
// 先读取
const width = el.offsetWidth;
const height = el.offsetHeight;

// 再写入
requestAnimationFrame(() => {
  el.style.cssText = `
    width: 100px;
    height: 200px;
  `;
});

/* ✅ 使用 transform 替代位置属性 */
.el {
  transform: translateX(100px); /* 代替 left: 100px */
}

CSS Containment

/* 限制样式作用范围，优化重排/重绘 */
.component {
  contain: layout;        /* 内部布局不影响外部 */
  contain: paint;         /* 内部绘制不影响外部 */
  contain: size;          /* 尺寸不依赖子元素 */
  contain: style;         /* 内部计数器等不穿透 */
  contain: content;       /* layout + paint */
  contain: strict;        /* layout + paint + size */
}

  contain-intrinsic-size: 300px; /* 预估尺寸，防止跳动 */
}

Q: 不定宽高元素如何居中？

不定宽高元素居中需要满足：

  1. Flex/Grid：天然支持，无需知道子元素尺寸
  2. 绝对定位 + transform：translate(-50%, -50%) 百分比基于自身尺寸
  3. 绝对定位 + margin: auto：子元素需设置 display: flex/table（形成 BFC）

避免：margin-top: -50%（百分比基于父元素宽度，不是高度）

层叠层 (@layer) Cascade Layers

解决的问题

CSS 选择器优先级战争：第三方库、工具类、组件样式、页面样式之间的优先级管理困难。层叠层提供了一种优先级分组机制，层与层之间按顺序比较，不受选择器特异性影响。

基础用法

/* 1. 定义层顺序（推荐在文件开头） */
@layer reset, base, components, utilities;

/* 2. 在指定层写样式 */
@layer reset {
  * { margin: 0; padding: 0; }
}

@layer components {
  .button { padding: 8px 16px; }
}

@layer utilities {
  .p-0 { padding: 0 !important; } /* 层内可用 !important */
}

/* 3. 匿名层（自动排在最后） */
@layer {
  /* 匿名层 */
}

/* 4. 非层样式（最高优先级） */
/* 未包裹在 @layer 中的样式 */

优先级顺序（从低到高）

  1. @layer 定义的第一层（如 reset）
  2. @layer 定义的后续层
  3. 非 @layer 样式（普通 CSS）
  4. 层内的 !important（反转顺序，第一层 !important 最高）
  5. 非层样式的 !important
  6. 内联样式（style 属性）
  7. 内联样式的 !important
最佳实践：
  • 框架/库样式放在低优先级层
  • 工具类/覆盖样式放在高优先级层
  • 避免在非层样式中写业务代码，保持可预测

:has() 父选择器 2022+

终于能选父元素了

/* 选择包含 img 的 card */
.card:has(img) {

}

/* 选择包含特定子元素的父元素 */
.nav:has(.is-active) {
  background: #f0f0f0;
}

/* 选择被 hover 的子元素影响的其他子元素（纯 CSS 手风琴） */
.accordion:has(.item:hover) .item:not(:hover) {

}

form:has(input:invalid) .submit {
  opacity: 0.5;
  pointer-events: none;
}

/* 配合 not */
.card:not(:has(img)) {
  /* 没有图片的卡片 */
}
