基础类型系统
Type System
原始类型 & 字面量类型
类型示例要点 / 易混
stringlet s: string = 'hi'小写 string（TS 类型）≠ 大写 String（JS 包装对象构造器）。永远用小写。
numberlet n: number = 42整数/浮点统一。NaN 和 Infinity 均为 number 类型。
booleanlet b: boolean = true不接受 0/'' 作为 boolean（与 JS 隐式转换不同）。
bigintlet big: bigint = 100n需 target ≥ ES2020；不能与 number 混合运算。
symbolconst s = Symbol('id')symbol vs unique symbol：后者是单例类型，const 才能推断为 unique symbol。
null / undefinedlet x: null = nullstrictNullChecks=false 时，二者是所有类型的子类型（可赋给任意类型）。开启后须显式处理。
字面量类型let d: 'GET' | 'POST'字符串/数字/布尔字面量均可作为类型。as const 推断字面量。
模板字面量类型`${string}-${number}`TS 4.1+，可用类型参数组合字符串模式，类型安全的字符串模板。
特殊类型完整对比 ★ 高频 难点
类型含义赋值能力典型用途关键区别
any关闭类型检查可赋给任何类型，接受任何类型迁移旧代码、紧急绕过类型系统"黑洞"，传染性强（any op = any）
unknown类型安全的 any只能赋给 unknown/any；读取前须收窄外部输入、JSON.parse 返回值必须类型守卫后才能使用，不传染
never不可达类型（空集）可赋给任何类型；无值能赋给它穷举检查、死代码、抛错函数返回联合类型中被自动去除；条件类型的底部
void函数无有效返回值可赋 undefined（strictNullChecks 下）函数返回类型标注void 函数可以有 return（返回 undefined）；never 函数永不返回
object非原始值类型接受对象/数组/函数（排除原始值）极少直接使用{}（空对象类型）≠ object：{} 接受一切非 null/undefined
{}有属性的对象（空约束）接受所有非 null/undefined（包括原始值）不推荐直接用与 object 的微妙区别：原始值字符串符合 {} 但不符合 object
⚠️
any vs unknown 核心区别：any 让你对值做任何操作，错误推迟到运行时；unknown 强制你在使用前做类型检查，错误在编译期暴露。处理外部数据（API响应、JSON.parse、用户输入）应始终用 unknown，不用 any。

联合类型 & 交叉类型精解 ★ 高频
联合类型（Union）A | B
  • 值是 A 或 B（或两者）；类型"或"关系
  • 只能访问共有成员（交集属性）
  • 使用前须收窄（typeof/instanceof/in/字面量）
  • 可辨识联合：共有字面量字段作为标签
  • 分布式特性：条件类型中联合各成员独立计算
    • type Shape = | { kind: 'circle'; r: number } | { kind: 'rect'; w: number; h: number }; // kind 是辨识字段
    交叉类型（Intersection）A & B
  • 值同时满足 A 和 B；类型"与"关系
  • 拥有所有成员（并集属性）
  • 同名属性类型冲突 → never（两个原始类型相交）
  • 常用于 Mixin 模式合并多个接口
  • 与 interface extends 的区别：& 同名属性取交叉（可能 never），extends 同名类型必须兼容
    • type AB = { a: string } & { b: number }; // { a: string; b: number } type X = string & number; // never
    satisfies 操作符详解 ★ TS4.9+ 新
    satisfies vs as vs 类型注解
    type Colors = Record<string, string | number[]>; // ❌ 类型注解：丢失字面量信息 const p1: Colors = { red: '#f00' }; p1.red.toUpperCase(); // Error: string | number[] 无 toUpperCase // ✅ satisfies：验证结构但保留具体类型 const p2 = { red: '#f00', blue: [0,0,255] } satisfies Colors; p2.red.toUpperCase(); // ✅ TS知道red是string p2.blue.map(x => x); // ✅ TS知道blue是number[]

    条件类型 & infer ★ 高频 难点
    条件类型基础
    // T extends U ? X : Y type IsString<T> = T extends string ? 'yes' : 'no'; // 分布式条件类型（naked type parameter） type ToArray<T> = T extends any ? T[] : never; type R = ToArray<string | number>; // string[] | number[]（各自计算后合并） // 阻止分布：包裹 tuple type NoDistrib<T> = [T] extends [any] ? T[] : never; type R2 = NoDistrib<string | number>; // (string | number)[]（整体计算）
    infer 类型推断
    // infer 在 extends 子句中捕获类型 type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never; type Awaited<T> = T extends Promise<infer U> ? Awaited<U> : T; // 推断函数参数类型 type FirstArg<T> = T extends (infer A, ...any[]) => any ? A : never; // TS5.4：infer 中使用 extends 约束 type Str<T> = T extends { val: infer V extends string } ? V : never;
    内置工具类型实现原理 ★ 高频
    工具类型实现原理作用
    Partial<T>{ [K in keyof T]?: T[K] }所有属性变为可选
    Required<T>{ [K in keyof T]-?: T[K] }所有属性变为必选（-? 去除可选）
    Readonly<T>{ readonly [K in keyof T]: T[K] }所有属性只读（浅层）
    Record<K,V>{ [P in K]: V }构建键值类型的字典
    Pick<T,K>{ [P in K]: T[P] }（K extends keyof T）从 T 中挑选属性 K
    Omit<T,K>Pick<T, Exclude<keyof T, K>>从 T 中排除属性 K
    Exclude<T,U>T extends U ? never : T（分布式）从联合 T 中排除可赋给 U 的类型
    Extract<T,U>T extends U ? T : never（分布式）从联合 T 中提取可赋给 U 的类型
    NonNullable<T>T extends null | undefined ? never : T去除 null 和 undefined
    ReturnType<T>T extends (...args: any) => infer R ? R : any获取函数返回类型
    Parameters<T>T extends (...args: infer P) => any ? P : never获取函数参数元组类型
    InstanceType<T>T extends new (...args: any) => infer R ? R : any获取构造函数实例类型
    Awaited<T>递归解包 Promise（TS4.5+）获取 Promise 解析后的类型
    NoInfer<T>阻止 T 参与类型推断（TS5.4+）防止推断位置影响泛型推断结果

    类型守卫全览 ★ 高频
    内置收窄手段
    // typeof 守卫 if (typeof x === 'string') { x.toUpperCase(); } // instanceof 守卫 if (err instanceof Error) { err.message; } // in 操作符 if ('swim' in animal) { animal.swim(); } // 等值收窄 if (x === null) { /* x is null */ } // 真值收窄（排除 falsy） if (x) { /* x 非 null/undefined/0... */ } // 赋值收窄 let x: string | number; x = 'hello'; // x: string
    自定义类型守卫（is）
    // 返回 paramName is Type 的谓词 function isString(v: unknown): v is string { return typeof v === 'string'; } function isUser(v: unknown): v is User { return ( typeof v === 'object' && v !== null && 'name' in v && 'age' in v ); } // 使用：收窄后安全访问 if (isUser(data)) { data.name; } // assertion functions（抛出而非返回false） function assert(v: unknown): asserts v is string { if (typeof v !== 'string') throw new Error(); }
    可辨识联合 & 穷举 ★ 高频
    type Shape = | { kind: 'circle'; r: number } | { kind: 'rect'; w: number; h: number } | { kind: 'tri'; base: number; height: number }; function area(s: Shape) { switch (s.kind) { case 'circle': return Math.PI * s.r**2; case 'rect': return s.w * s.h; case 'tri': return s.base * s.height / 2; default: // 穷举检查 const _: never = s; // 漏掉分支→编译错误 throw new Error(); } }
    类型推断规则 & 上下文类型
    推断规则
    • 最宽泛推断：let x = [1, 'a'] → (string | number)[]
    • 最佳公共类型：多个候选类型取兼容的上层类型
    • const 推断字面量：const x = 'GET' → 'GET'（不是 string）
    • 函数返回类型：自动推断，所有 return 分支的联合
    • 泛型推断：从实参推断类型参数，可部分手动指定
    • 条件类型延迟：T 为裸类型参数时不立即求值
    上下文类型（Contextual Typing）
    // 回调参数从上下文推断类型 [1,2,3].map(x => x * 2); // x: number // 对象字面量上下文推断 const handler: EventHandler = { onClick(e) { /* e 被推断 */ } }; // 函数赋值给有类型的变量 const f: (x: number) => string = x => x.toString(); // x: number，返回推断 string // ⚠️ 独立箭头函数不从上下文推断 const g = x => x * 2; // Error：x 隐式 any

    Class & Decorators
    类修饰符全表 ★ 高频
    修饰符访问范围继承要点
    public任何地方✅默认修饰符，可省略
    private仅本类❌（编译时）TS 语法私有，编译为 JS 后消失；子类不可访问
    #field（JS 私有）仅本类❌（运行时）真正私有，运行时强制；反射/代理也无法访问
    protected本类 + 子类✅外部不可访问，子类可访问
    readonly--只能在声明或 constructor 中赋值
    static类本身✅（子类继承静态）实例无法访问；可 static #x 组合私有静态
    abstract-必须覆写抽象类不可实例化；抽象方法无实现
    override--TS4.3+；显式标记覆写父类方法，父类无该方法则报错
    装饰器（Decorators）★ TS5.0 正式版

    运行时类型验证
    TS 类型只在编译期存在，运行时无类型信息。外部数据必须验证：
    • Zod：z.string().parse(data) — schema 验证 + 类型推断，最流行
    • Valibot：模块化，更小体积
    • io-ts：函数式风格，基于 Either
    • class-validator：装饰器风格，NestJS 集成
    • 手写守卫：轻量场景，is 类型谓词
    import { z } from 'zod'; const User = z.object({ name: z.string(), age: z.number() }); type User = z.infer<typeof User>; // 自动推断类型
    类型体操实践原则
    • 优先用简单类型；避免过度工程化
    • 类型复杂度与代码复杂度成正比，要合理权衡
    • 使用 type-fest、ts-toolbelt 等库而非自造轮子
    • 复杂类型加注释说明意图
    • 避免将业务逻辑放入类型系统（类型体操≠类型安全）
    • // @ts-expect-error 优于 // @ts-ignore（前者在错误消失后报错，避免遗忘）

    高频易混淆对比 ★ 必背
    概念 Avs概念 B核心差异
    interfacevstype interface 可声明合并（同名自动合并）；type 可表达联合/条件/映射类型。对象类型两者等价，优先 interface 用于公共 API（可扩展），type 用于复杂类型计算
    anyvsunknown any 关闭检查，可直接操作，传染性强；unknown 类型安全，必须收窄才可操作，不传染。外部输入用 unknown。
    nevervsvoid never：函数永不返回（抛错/死循环）；void：函数没有有意义的返回值（可以 return undefined）。never 是所有类型的子类型（赋给任何类型）
    type A = B | Cvstype A = B & C 联合（|）：A 或 B，只能访问共有属性；交叉（&）：同时满足，拥有全部属性。同名原始类型相交=never
    Exclude<T,U>vsOmit<T,K> Exclude 操作联合类型，排除可赋给 U 的成员；Omit 操作对象类型，排除指定属性键。常混淆：Exclude 用于 type A|B|C，Omit 用于 interface
    typeofvskeyof typeof 获取值的类型（typeof obj → 对象类型）；keyof 获取类型的键联合（keyof Obj → 'a'|'b'|'c'）。组合：keyof typeof obj 获取对象键的联合
    asvssatisfies as 强制断言（可能不安全，丢失推断）；satisfies 验证兼容性但保留原始字面量类型推断（两全其美，TS4.9+）
    private（TS）vs#field（JS） TS private：仅编译期检查，JS 输出后变为普通属性；#field：运行时强制私有，完全不可访问，reflect/proxy 也无法穿透
    T extends U（约束）vsT extends U（条件） 泛型约束 <T extends U>：限制 T 必须兼容 U；条件类型 T extends U ? X : Y：判断 T 是否可赋给 U。语法相同，位置不同含义不同
    enumvsconst enum enum 生成运行时代码（双向映射对象）；const enum 编译内联（零运行时），但 isolatedModules=true 时跨文件使用受限。推荐用 as const 对象替代
    import typevsimport import type 纯类型导入，编译后完全删除（用于 isolatedModules 场景，避免副作用）；import 可能保留副作用。verbatimModuleSyntax=true 时强制区分
    declare modulevsdeclare namespace declare module 扩充已有模块类型（用于为第三方库添加类型）；declare namespace 定义命名空间类型（旧式或全局变量用）
    面试高频原理题速查 ★ 必背
    问题核心答案
    TS 类型系统是健全的吗 不完全健全（Unsound）。设计决策：any/类型断言/函数双变参数（strictFunctionTypes 前）/协变数组等允许不安全操作。权衡了实用性与安全性。
    如何实现类型安全的 EventEmitter 泛型事件 Map + 模板字面量类型：on<K extends keyof Events>(event: K, handler: (data: Events[K]) => void)，事件名和参数全部类型安全。
    分布式条件类型何时触发 条件类型中 T 为裸类型参数（未被包裹）且传入联合类型时，自动对每个成员分发计算。用 [T] 包裹可阻止分发。
    逆变在函数类型中的表现 函数参数位置逆变：接受父类的函数可赋给接受子类的函数类型（反直觉）。strictFunctionTypes 开启后方法属性（非简写）强制逆变。
    如何实现 DeepReadonly 递归映射类型：{ readonly [K in keyof T]: T[K] extends object ? DeepReadonly<T[K]> : T[K] }。注意函数/数组/Date 等 object 子类型需特殊处理。
    infer 的本质是什么 在 extends 子句的模式匹配中声明临时类型变量，捕获被匹配位置的类型。类似于正则捕获组，只在条件类型的 true 分支可用。
    结构类型和名义类型的区别 TS 用结构类型：只要形状（属性/方法）匹配即兼容，无需同名。可用 brand（品牌类型）模拟名义类型：type UserId = string & { _brand: 'UserId' }。
    noUncheckedIndexedAccess 的作用 数组下标访问返回 T | undefined，强制处理越界情况。提升数组操作安全性，但需大量 ?? / 判断，要权衡代码量。
