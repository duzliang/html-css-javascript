语义化元素与文档结构
Semantics
结构元素对比
元素语义典型场景常见误用
<header> 页面或区块的头部，可多次使用 站点顶部 nav、article 标题区 只用一次，误以为=<head>
<nav> 主导航链接组 顶部/侧栏导航，面包屑 页脚版权链接也套 nav
<main> 页面主体内容，每页唯一 内容区域，跳过导航 landmark 放在 article/section 内
<article> 独立可分发内容单元 博客文章、评论、新闻 与 section 混淆
<section> 按主题分组内容，通常含标题 文章章节、功能板块 当 div 用，无实际主题
<aside> 与主内容相关但可独立的内容 侧边栏、广告、相关链接 放在 article 内做注释区
<footer> 区块或页面的页脚 版权、联系、文章元数据 与 <address> 混淆
<figure> / <figcaption> 独立的自含内容（图、代码、表格） 图文说明、代码示例 只用于图片，忽略代码/表格场景
<details> / <summary> 可折叠的披露控件，原生 accordion FAQ、折叠说明 不知道可用 JS 控制 open 属性
<dialog> 模态/非模态对话框，原生弹窗 确认框、表单弹窗 直接用 div 模拟，忽略焦点管理
<mark> 与上下文相关的高亮文本 搜索词高亮 用 span+style 代替
<time> 机器可读的时间/日期 文章发布时间，datetime 属性 直接输出文字，不加 datetime
<address> 联系信息，属于最近祖先 article/body 作者邮箱、公司地址 当通用地址标签用
<picture> 响应式图片，多源条件切换 不同屏幕尺寸/格式的图 只用 img srcset，忽略 type 格式切换
内联文本语义
strong vs b / em vs i
  • <strong> = 重要性（语义），<b> = 视觉加粗（无语义权重）
  • <em> = 强调/语气（影响读音），<i> = 专业术语/外语/内心独白
  • 屏幕阅读器会读出 em/strong 的语气差异
del / ins / s
  • <del> 表示已删除内容（文档编辑），可附 cite/datetime
  • <ins> 表示插入的新内容，与 del 配对
  • <s> 表示不再准确/过时（如原价），非删除操作
abbr / dfn / cite / q / blockquote
  • <abbr title="..."> 缩写，hover 提示全称
  • <dfn> 定义项，首次引入术语
  • <cite> 作品标题引用（书、电影、论文）
  • <q> 行内引用，自动加引号；<blockquote cite> 块级引用
WAI-ARIA 与语义化协同 ★ 高频
💡
第一原则：优先使用原生语义元素。 若必须用 div/span，才补 ARIA role。
role="button" + tabindex="0" + 键盘事件 ≠ 真正的 <button>（原生有焦点管理、Enter/Space、禁用态等）。
Landmark roles（地标）
  • banner → <header> （顶层）
  • navigation → <nav>
  • main → <main>
  • complementary → <aside>
  • contentinfo → <footer> （顶层）
  • region + aria-label → <section>
  • search → 搜索表单（新增 role）
关键 ARIA 属性
  • aria-label / aria-labelledby — 控件可访问名称
  • aria-describedby — 补充描述，如错误提示
  • aria-live="polite|assertive" — 动态内容播报
  • aria-expanded — 折叠/展开状态
  • aria-hidden="true" — 从 AT 树中移除
  • aria-current="page" — 当前导航项
  • aria-busy="true" — 加载中状态

约束验证 API ★ 高频
内置验证属性
  • required — 必填
  • pattern="[regex]" — 正则匹配
  • minlength / maxlength — 字符数
  • min / max — 数值/日期范围
  • step — 步进，影响有效性
  • novalidate（form 属性）— 关闭内置验证
  • formnovalidate（submit 按钮）— 仅此提交跳过验证
Constraint Validation API
  • el.validity — ValidityState 对象（valueMissing / typeMismatch / patternMismatch / tooLong / rangeOverflow 等）
  • el.checkValidity() — 触发验证，返回 bool
  • el.reportValidity() — 验证并显示气泡提示
  • el.setCustomValidity("msg") — 自定义错误（空字符串=清除）
  • :valid / :invalid / :user-invalid CSS 伪类
表单新特性
<datalist>
为 input 提供可搜索的建议列表（非强制选择），通过 list 属性关联。与 <select> 的本质区别：用户可输入任意值。
<output>
表示计算结果。for 属性关联输入控件，ARIA role=status，适合实时计算展示（如 range + output 显示当前值）。
autocomplete 精细控制
autocomplete="new-password" 阻止密码管理器自动填充。"cc-number" 提示信用卡。"one-time-code" 短信验证码自动填充（OTP）。"off" 已基本被忽略。
inputmode 属性
控制移动端软键盘类型，独立于 type：inputmode="numeric"（纯数字）、"decimal"（含小数点）、"tel"、"email"、"url"、"search"、"none"（不弹键盘）。
enterkeyhint 属性
控制移动端回车键显示文字："search"、"send"、"go"、"done"、"next"、"previous"。提升移动端表单体验。
form 关联属性
form="formId" 使控件关联非父级表单（实现跨位置提交）。formaction / formmethod / formenctype 在 submit 按钮上覆盖 form 设置，实现"保存草稿"与"正式提交"差异化。

核心浏览器 API
APIs
通信 API 对比 ★ 高频
API方向协议适用场景关键点
XMLHttpRequest双向HTTP 兼容旧代码 事件驱动，回调地狱，支持进度事件 progress
Fetch API双向HTTP 现代 Ajax Promise，需手动处理 404（非 reject），AbortController 取消，默认不带 cookie（credentials: 'include'）
WebSocket全双工ws:// 实时推送、聊天、游戏 握手后保持连接，二进制/文本帧，心跳 ping/pong
SSE (EventSource)服务器→客户端HTTP 股价推送、通知流 自动重连，支持事件命名/id，HTTP/2 下无连接数限制
WebRTCP2P 全双工DTLS/SRTP 视频通话、屏幕共享 ICE/STUN/TURN 穿透，DataChannel 传数据
BroadcastChannel同源标签间- 多标签页同步状态 同源限制，postMessage 类 API，无 SharedWorker 复杂
Worker 体系对比 ★ 高频
类型数量生命周期典型用途通信
Web Worker多实例页面控制 CPU 密集计算（图像处理、加密） postMessage，可 Transferable（零拷贝）
SharedWorker同源共享一个所有关联页面 多标签共享连接（如 WebSocket） MessagePort，需 port.start()
Service Worker域级单例独立于页面 离线缓存、推送、请求拦截 postMessage、Cache API、Fetch 事件
Worklet-浏览器控制 CSS Paint/Layout/Audio 扩展 无通信，轻量沙箱
Observer API 族
IntersectionObserver
检测元素与视口（或祖先）的交叉比例。threshold 设触发点，rootMargin 提前触发。适合：懒加载、曝光埋点、无限滚动。

⚠️ isIntersecting 与 intersectionRatio 区别：ratio=0 时 isIntersecting 可能为 true（仅边缘接触）。
MutationObserver
监听 DOM 变化（子树、属性、文本）。subtree:true 深度监听。disconnect() 主动断开防泄漏。适合：第三方内容监控、实现撤销/重做。

⚠️ 回调在微任务队列执行（Promise 之后、宏任务之前）。
ResizeObserver
监听元素尺寸变化（非 window.resize）。borderBoxSize / contentBoxSize / devicePixelContentBoxSize 三种尺寸。适合：响应式图表、自适应组件。

⚠️ 注意循环触发（修改尺寸又触发 resize）。
PerformanceObserver
订阅性能条目流（paint、navigation、resource、longtask、largest-contentful-paint、layout-shift）。替代 performance.getEntries()。

buffered:true 获取历史条目，适合 LCP/CLS 测量。
重要 API 速查
History API
  • pushState(state, '', url) — 不刷新修改 URL
  • replaceState — 替换当前历史条目
  • popstate 事件 — 前进/后退触发
  • SPA 路由基础；注意 state 有 640KB 限制
Clipboard API
  • navigator.clipboard.writeText() — 写文本
  • navigator.clipboard.write([ClipboardItem]) — 写富内容（图片）
  • navigator.clipboard.readText() — 需用户授权
  • 仅 HTTPS + focus 状态可用
File System Access API
  • showOpenFilePicker() 打开文件
  • showSaveFilePicker() 另存为
  • showDirectoryPicker() 目录访问
  • Chrome 主导，Firefox 不支持（截止2024）
Geolocation API
  • navigator.geolocation.getCurrentPosition()
  • watchPosition() — 持续追踪
  • HTTPS 必须；coords 含 accuracy 精度
  • timeout / maximumAge 选项控制缓存
Drag and Drop API
  • draggable="true" 启用
  • 事件：dragstart / drag / dragenter / dragover / dragleave / drop / dragend
  • dataTransfer.setData() / getData()
  • dragover 必须 preventDefault() 才能触发 drop
Web Share API
  • navigator.share({ title, text, url, files })
  • 调用系统级分享菜单
  • 需用户手势触发，HTTPS 必须
  • navigator.canShare() 先检测能力
新兴 API（2023-2025）
View Transitions API ★ 新
原生页面/状态切换动画。document.startViewTransition(callback) 自动截图新旧状态并过渡。CSS 中用 ::view-transition-old 和 ::view-transition-new 定制动画。支持 MPA（跨页面）模式（需 opt-in）。
Popover API ★ 新
原生弹出层。popover 属性 + popovertarget 无需 JS 即可控制。popover="auto"（点外关闭，互斥）vs "manual"（手动控制）。showPopover() / hidePopover() / togglePopover() JS API。Top Layer 渲染，自动焦点管理。
Speculation Rules API ★ 新
预测性预渲染。<script type="speculationrules"> 定义预取/预渲染规则。浏览器在空闲时后台渲染目标页，实现"即时跳转"。比 <link rel="prefetch"> 更激进（完整渲染）。
Navigation API ★ 新
替代 History API 的现代路由基础。navigation.navigate() / navigation.back()。navigate 事件可拦截并处理转场。解决了 popstate 无法区分前进/后退的问题。

Storage
方案容量生命周期线程适用场景关键限制
Cookie~4KB/条expires/session主线程（document.cookie） 会话认证、服务端读取 每次请求携带，SameSite/HttpOnly/Secure，第三方 Cookie 已逐步淘汰

StorageManager API

Media & Graphics
图片响应式方案对比 ★ 高频
方案用途浏览器选择权代码示例要点
img srcset + sizes同内容不同分辨率浏览器自选最优 srcset="img-1x.jpg 1x, img-2x.jpg 2x" 或 srcset="img-400.jpg 400w" sizes="(max-width:600px) 100vw, 50vw"

现代图片格式
WebP
Google 研发，支持有损/无损/动画/透明。比 JPEG 小 25-34%，比 PNG 小约 26%。全浏览器支持（IE 除外）。
AVIF
基于 AV1 编码，压缩率更高（比 WebP 再小 20%+）。支持 HDR、宽色域。编码慢（适合预生成）。Chrome/Firefox 支持，Safari 16+。
SVG
矢量无损。内联 SVG 可 CSS/JS 控制。currentColor 继承颜色。viewBox 控制坐标系。use 元素复用。SMIL 动画（逐渐废弃）→ CSS 动画代替。
Canvas vs WebGL vs SVG
技术渲染适合场景性能特点
Canvas 2D像素（位图）图表、截图、滤镜处理单线程，OffscreenCanvas 可 Worker
WebGL (1.0/2.0)GPU 着色器3D、粒子、数据可视化GPU 并行，着色器语言 GLSL
WebGPU现代 GPU API复杂 3D、AI 推理、通用计算比 WebGL 更底层，支持 Compute Shader
SVG矢量（DOM）图标、可交互图形、数据可视化可 CSS/JS 动画，节点多时性能下降
CSS Houdini自定义渲染自定义背景、布局、动画Paint/Layout/Animation Worklet

性能优化 & 资源加载
Performance
资源提示（Resource Hints）全对比 ★ 高频
rel 值作用是否执行跨域典型用法
dns-prefetch预解析 DNS否✅ CDN 域名、第三方服务
preconnectDNS+TCP+TLS 握手否✅（需 crossorigin） 字体服务、API 域名（8s 内必用）
prefetch空闲时获取未来页面资源否（脚本不执行）✅ 下一步可能的页面、懒加载资源
preload高优先级获取当前页资源否（需 as 属性）同源优先 LCP 图片、关键字体、首屏 JS
modulepreload预加载+解析 ES Module解析（不执行）CORS ES Module 依赖树预热
⚠️
preload 陷阱：必须配合 as 属性（font/image/script/style/fetch），否则优先级失效且产生重复请求。字体 preload 必须加 crossorigin（即使同源）。未在页面实际使用的 preload 资源会触发 Chrome 警告。
Script 加载策略对比 ★ 高频
属性下载执行时机顺序保证适用
（无）阻塞 HTML 解析立即执行✅遗留代码
async并行下载下载完立即执行（可能乱序）❌独立无依赖脚本（统计、广告）
defer并行下载DOM 解析完成后，DOMContentLoaded 前✅需 DOM 的脚本，推荐默认方案
type="module"并行下载等同 defer + 严格模式 + 独立作用域✅ES Module
type="module" async并行下载下载完立即执行❌独立 ES Module
Web Vitals & 性能指标
LCP（最大内容绘制）
<2.5s 为优。候选元素：img、video poster、背景图、块级文本。优化：fetchpriority="high" 图片、preload LCP 图、SSR 直出、CDN。
INP（交互到下一帧）
替代 FID（2024 年 3 月）。<200ms 为优。测量所有交互的响应延迟。优化：长任务拆分（scheduler.yield）、避免主线程阻塞、减少 Input Delay。
CLS（累积布局偏移）
<0.1 为优。图片/视频设 width/height 避免跳变。字体用 font-display:optional。动态内容预留空间。transform 动画不触发 layout shift。
lazy loading & fetchpriority
loading="lazy"
原生图片/iframe 懒加载。浏览器自动计算阈值（视口距离）。
⚠️ LCP 图片禁止加 lazy（会延迟最重要图片）。
⚠️ 首屏内图片不需要 lazy，浏览器自动调优。
fetchpriority 属性
fetchpriority="high|low|auto"，用于 img/link/script/fetch。
LCP 图片设 high 可显著提升加载优先级。low 用于折叠下图片。

Subresource Integrity & Trusted Types
Subresource Integrity (SRI)
<script src="cdn/lib.js" integrity="sha384-xxxx" crossorigin="anonymous">

浏览器验证资源哈希，防 CDN 被篡改。crossorigin 必须设置（CORS 预检）。可同时列多个哈希算法。
Trusted Types API
通过 CSP require-trusted-types-for 'script' 强制要求所有 sink（innerHTML/eval/script.src 等）接受 TrustedHTML/TrustedScript 对象，从源头消除 DOM XSS。Chrome 支持，Firefox 实验性。
隐私相关 & 第三方 Cookie 替代
Privacy Sandbox

Sanitizer API
浏览器原生 XSS 净化。el.setHTML(dirty, { sanitizer })，可配置允许/禁止的标签和属性。替代 DOMPurify 的未来方向（规范仍在演进）。

Spec & Gotchas
DOCTYPE & 解析模式 ★ 必知
解析模式
  • Quirks Mode — 无 DOCTYPE 或 HTML4 怪异 DOCTYPE，模拟旧浏览器
  • Limited Quirks — 部分旧 DOCTYPE 触发
  • Standards Mode — <!DOCTYPE html> 触发，现代标准
  • 差异：盒模型（IE box-sizing）、表格样式、行高计算
HTML vs XHTML vs HTML5
  • XHTML：严格 XML 语法，自闭合标签必须 />，属性必须引号
  • HTML5：容错解析，Living Standard（WHATWG 维护，无版本号）
  • text/html → HTML 解析器；application/xhtml+xml → XML 解析器（严格）
常见易混淆对比 ★ 高频
概念Avs概念B核心差异
innerHTMLvstextContent / innerText innerHTML 解析 HTML（XSS 风险）；textContent 纯文本（性能好）；innerText 感知 CSS（visibility:hidden 内容会被忽略，触发 reflow）
attributevsproperty attribute 是 HTML 初始值（字符串，getAttribute）；property 是 DOM 当前状态（类型丰富）。checked attribute 是默认值，checked property 是当前值
document.querySelectorvsgetElementById 等 querySelector 返回静态 Node；getElementsBy* 返回动态 HTMLCollection（DOM 变化实时反映）
event.targetvsevent.currentTarget target = 实际触发元素；currentTarget = 绑定监听器的元素（在捕获/冒泡过程中变化）
stopPropagationvsstopImmediatePropagation stopPropagation 阻止事件冒泡/捕获；stopImmediatePropagation 还阻止同元素上其他监听器
DOMContentLoadedvsload DCL: DOM 解析完成（CSS/图片未必加载完）；load: 所有资源（图片、样式表、iframe）加载完毕
clientWidthvsoffsetWidth / scrollWidth / getBoundingClientRect clientWidth = 内容+padding（不含边框、滚动条）；offsetWidth = 内容+padding+border；scrollWidth = 含溢出内容；getBCR = 含 transform 的视口坐标
<link rel="stylesheet">vs@import link 并行加载；@import 串行（在 CSS 解析阶段触发）；link 可被 JS 动态控制；@import 不支持媒体查询（仅支持媒体类型）
meta viewportvs@viewport（废弃） <meta name="viewport" content="width=device-width,initial-scale=1"> 是标准；initial-scale=1 防 iOS 自动缩放；user-scalable=no 损害无障碍，不推荐
tabindex="0"vstabindex="-1" 0 = 加入 tab 序（顺序由 DOM 决定）；-1 = 不在 tab 序但可 focus()（用于 JS 控制焦点）；正数（1+）= 优先 tab，不推荐（破坏顺序）
HTML 解析容错 & 陷阱
可选标签 & 隐式结构
  • HTML5 允许省略 <html>、<head>、<body>，浏览器自动补全
  • <p> 遇到块级元素自动关闭（隐式关闭）
  • <table> 外的文本会被浏览器移到表格之前（foster parenting）
  • <li> 不需要关闭标签（遇到下一个 li 自动关闭）
字符实体 & 编码
  • 推荐直接 UTF-8 + <meta charset="UTF-8">，减少实体使用
  • 必须转义：&amp; &lt; &gt; &quot; &apos;
  • 不可见字符：&nbsp;（不折行空格）、&shy;（软连字符）
  • &#8203; 零宽断行符，避免长单词溢出

面试高频原理题速查
问题核心答案
从输入 URL 到页面展示 DNS解析→TCP连接→TLS握手→HTTP请求→服务器响应→HTML解析（DOM）→CSS解析（CSSOM）→渲染树→Layout→Paint→Composite；JS阻塞解析；资源并行加载
重排（Reflow）vs 重绘（Repaint） Reflow：几何变化，影响布局，代价最高；Repaint：外观变化，不影响布局；只改 transform/opacity 仅走 Composite，最廉价
script async vs defer async：下载完立即执行，不保证顺序；defer：DOM 解析完执行，保序，推荐默认方案
什么是跨源隔离 COOP: same-origin + COEP: require-corp 开启，允许 SharedArrayBuffer，防 Spectre 攻击
meta charset 必须在 head 最前 浏览器在读取 1024 字节内必须确定编码，charset 必须在文件前 1KB 且在任何含非 ASCII 内容之前
HTML 解析遇到 script 为何阻塞 脚本可能调用 document.write 修改 DOM，解析器无法预知，必须等脚本执行完才能继续（预加载扫描器优化了资源下载）
为什么 CSS 放 head，JS 放底部 CSS 放 head：避免 FOUC（无样式内容闪现），CSSOM 需在渲染前就绪；JS 放底部：避免阻塞 HTML 解析（现代推荐 defer 代替）
