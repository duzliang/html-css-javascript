语义化元素与文档结构
Semantics
▼
header vs head vs h1-h6
  • <header> - 页面或区块的头部，可多次使用
  • <head> - 页面元数据容器，不可见
  • <h1>-<h6> - 标题层级，每个页面应有唯一 h1
面试高频
🎯 快速测试
article vs section vs div
  • article - 独立可分发内容单元（博客文章、评论）
  • section - 按主题分组，通常含标题
  • div - 无语义，仅用于样式/脚本钩子
💡
判断技巧：article 可以独立出现在 RSS/聚合内容中；section 必须有主题；无主题用 div
面试高频
nav 的正确使用
  • 用于主导航链接组，非所有链接
  • 页脚版权链接、社交媒体链接不应用 nav
  • 页面可以有多个 nav（主导航、面包屑）
  • 配合 aria-label 区分多个导航
main 元素规范
  • 每页只能有一个 main 元素
  • 不能嵌套在 article、aside、header、nav 等中
  • 是 ARIA landmark，屏幕阅读器可一键跳转
  • IE 不支持，需要 polyfill
<!-- 正确用法 --> <body> <header>...</header> <main> <article>...</article> </main> <footer>...</footer> </body>
面试高频
details / summary - 原生折叠
  • 无需 JS 实现 accordion 效果
  • open 属性控制展开状态
  • 支持 toggle 事件监听
  • 可配合 CSS [open] 选择器定制样式
<details> <summary>点击展开</summary> <p>这是隐藏内容...</p> </details>
dialog - 原生模态框
  • showModal() - 模态，背景不可交互，自动焦点管理
  • show() - 非模态
  • close() - 关闭
  • ::backdrop 伪元素定制背景
  • ESC 键自动关闭（可阻止）
const dialog = document.querySelector('dialog'); dialog.showModal(); // 打开模态框
新特性
ARIA 与语义化协同
🎯
第一原则：优先使用原生语义元素。 若必须用 div/span，才补 ARIA role。
  • role="button" 需要配合 tabindex="0" 和键盘事件
  • 但永远无法完全模拟原生 button（焦点管理、禁用态等）
  • 常用 landmark: banner, navigation, main, complementary, contentinfo
  • aria-live="polite|assertive" - 动态内容播报
面试高频
picture 响应式图片
  • Art Direction: 不同尺寸显示不同裁剪
  • 格式降级: AVIF → WebP → JPEG
  • 最后一个 img 是必须的 fallback
<picture> <source type="image/avif" srcset="img.avif"> <source media="(max-width:600px)" srcset="mobile.jpg"> <img src="fallback.jpg" alt="描述"> </picture>

面试高频
Cookie 现代属性
  • SameSite=Strict - 跨站不发送（最强防 CSRF）
  • SameSite=Lax - Chrome 默认，顶级 GET 导航发送
  • SameSite=None; Secure - 第三方必须加 Secure
  • HttpOnly - 禁止 JS 读取（防 XSS）
  • __Host- 前缀 - 强制 HTTPS+Secure+Path=/

StorageManager API
  • navigator.storage.estimate() - 查询配额
  • navigator.storage.persist() - 申请持久化
  • navigator.storage.persisted() - 检查授权
Storage Partitioning
  • 2023+ 主流浏览器将三方 Cookie 分区
  • 按 (top-site, frame-site) 分区
  • iframe 中的 localStorage 与顶层隔离
