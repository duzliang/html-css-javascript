盒模型 & 视觉格式化模型
Box Model
box-sizing 对比 ★ 高频
content-box（W3C 默认）
  • width/height = 内容区，不含 padding/border
  • 实际占宽 = width + padding×2 + border×2
  • 动态计算容易出错，不推荐
width:200px + padding:20px + border:2px
→ 实占 244px
border-box（推荐）
  • width/height 包含 padding + border
  • 实际占宽 = 设定 width（直觉友好）
  • 全局 * { box-sizing: border-box } 最佳实践
width:200px + padding:20px + border:2px
→ 实占 200px
视觉格式化模型（BFC / IFC / FFC / GFC）★ 高频
BFC — 块级格式化上下文
触发条件：overflow≠visible、float、position:absolute/fixed、display:flow-root（最干净）、flex/grid 子项、contain:layout

效果：
  • 内部浮动元素参与高度计算（清除浮动）
  • 阻止外边距折叠（margin collapse）
  • 不被外部浮动元素覆盖（自适应两栏）
Margin Collapse（外边距折叠）
折叠条件（三种场景）：
  • 相邻兄弟元素：上下 margin 取最大值
  • 父子元素：子 margin-top 与父融合（无 border/padding/BFC 隔离时）
  • 空块：自身 margin-top + margin-bottom 折叠
阻止折叠：BFC、border、padding、display:flex/grid 的子项不折叠
Stacking Context（层叠上下文）★ 高频
触发条件：root、position+z-index≠auto、opacity<1、transform/filter/clip-path≠none、will-change、isolation:isolate、mix-blend-mode≠normal、contain:paint

层叠顺序（从下到上）：背景/边框 → 负z-index → 块级盒 → 浮动盒 → 行内盒 → z-index:0/auto → 正z-index
contain 属性
隔离元素对页面其余部分的影响，优化渲染性能：
  • contain:layout — 布局隔离，触发 BFC
  • contain:paint — 绘制裁剪，触发层叠上下文
  • contain:size — 尺寸隔离（不依赖内容尺寸）
  • contain:strict = layout + paint + size
  • content-visibility:auto — 视口外元素跳过渲染
display 值对照
display生成盒类型特点典型元素
block块级盒独占一行，可设宽高，margin/padding 全生效div, p, h1-h6
inline行内盒不独占行，宽高无效，水平 margin/padding 生效，垂直不影响行高span, a, strong
inline-block行内块行内排列但可设宽高，baseline 对齐img, button（默认）
flex弹性容器（块级）一维布局，主轴/交叉轴模型现代布局主力
inline-flex弹性容器（行内）行内排列的 flex 容器行内按钮组
grid网格容器（块级）二维布局，行列同时控制页面整体结构
contents无盒（穿透）自身不生成盒，子元素提升，常用于 flex/grid 穿透中间包装层
flow-root块级盒 + 新 BFC清除浮动最佳方式，无副作用替代 overflow:hidden
table / table-cell…表格盒模拟表格布局（旧时代垂直居中）兼容旧方案
none无不在渲染树，不占空间，无障碍不可见切换隐藏

Layout
Flexbox 核心属性速查 ★ 高频
容器属性 (flex container)
属性值 / 说明
flex-directionrow | row-reverse | column | column-reverse
flex-wrapnowrap | wrap | wrap-reverse
justify-contentflex-start | center | flex-end | space-between | space-around | space-evenly
align-itemsstretch | flex-start | center | flex-end | baseline
align-content多行时控制行间距（同 justify-content 值集合）
gaprow-gap + column-gap，替代 margin hack
子项属性 (flex item)
属性值 / 说明
flex-grow放大比例，0=不放大，默认0
flex-shrink缩小比例，0=不缩小，默认1
flex-basis初始主轴尺寸，auto | 具体值
flex简写：flex:1=1 1 0%；flex:auto=1 1 auto；flex:none=0 0 auto
align-self覆盖容器 align-items
order排列顺序，默认0，可负数
⚠️
flex:1 陷阱：flex:1 = flex: 1 1 0%，flex-basis 为 0，内容最小尺寸被忽略。flex:auto = flex: 1 1 auto，flex-basis 为内容大小。若子项有最小宽度需求用 min-width:0 允许收缩（text overflow 常见原因）。
CSS Grid 核心概念 ★ 高频
轨道定义
  • grid-template-columns: repeat(3, 1fr) 等分三列
  • repeat(auto-fill, minmax(200px, 1fr)) 响应式自动列
  • auto-fill vs auto-fit：fill 保留空轨道，fit 折叠空轨道（影响最后行对齐）
  • fr 单位：可用空间的比例（先扣除固定宽度和 gap）
  • subgrid：子元素参与父网格对齐（CSS Grid Level 2）
位置控制
  • grid-column: 1 / 3 跨两列（或 span 2）
  • grid-area: header 命名区域放置
  • grid-template-areas 可视化布局定义
  • grid-auto-flow: dense 自动填充空洞
  • 隐式网格：grid-auto-rows / grid-auto-columns 控制尺寸
对齐属性（Grid vs Flex 对比）
  • justify-items — 单元格内水平对齐（Grid 独有）
  • align-items — 单元格内垂直对齐
  • justify-content — 整体网格水平位置
  • align-content — 整体网格垂直位置
  • place-items: center — align+justify 简写，两行居中神器
subgrid ★ 新
  • grid-template-columns: subgrid 继承父 grid 的列轨道
  • 解决卡片组件内部元素跨卡片对齐问题
  • Chrome 117+、Firefox 71+、Safari 16+ 支持
  • 典型用：表单标签+输入框多行对齐
定位体系 ★ 高频
position脱离文档流定位参考关键行为
static否-默认，top/left 无效，z-index 无效
relative否（占位）自身原始位置偏移但原空间保留；创建层叠上下文（z-index≠auto）
absolute是最近 position≠static 祖先无祖先则相对 initial containing block（视口）
fixed是视口transform/filter 祖先会改变包含块为该祖先！
sticky否（混合）滚动容器relative + fixed 混合；overflow:hidden 祖先会使其失效
⚠️
fixed 定位陷阱：祖先元素有 transform、filter、perspective、will-change:transform 时，fixed 子元素的包含块变为该祖先而非视口，导致"fixed 失效"。
⚠️
sticky 失效常见原因：父容器 overflow:hidden/auto/scroll、父容器高度不足无法滚动、未设定 top/left/right/bottom 阈值。
多列 & 传统布局
Multi-column Layout
  • column-count / column-width / columns 简写
  • column-gap、column-rule（分隔线）
  • column-span: all 元素跨所有列
  • break-inside: avoid 防止跨列断行
Float（遗留）
  • 脱离文档流，文字环绕
  • 清除浮动：display:flow-root（推荐）、clearfix hack、overflow:hidden
  • clear:both/left/right 在兄弟元素上使用
  • 现代布局中几乎只用于文字环绕图片
Table Layout
  • table-layout:fixed：列宽由第一行决定，性能更好
  • border-collapse:collapse vs separate
  • vertical-align 仅在 table-cell 中完全生效
  • 现代 display:table-cell 仍用于垂直居中兼容

选择器 & 层叠 & 继承
Cascade
优先级（Specificity）计算规则 ★ 高频
优先级权重
!important + 行内 最高（覆盖一切）
行内样式 style="" (1,0,0,0)
ID 选择器 #id (0,1,0,0)
类/.class 属性[attr] 伪类:hover (0,0,1,0)
元素 div 伪元素::before (0,0,0,1)
通配符 * 关系符 + > ~ (0,0,0,0)
特殊优先级规则
  • :is() / :not() / :has() 优先级 = 参数列表中最高选择器
  • :where() 优先级永远为 0（0,0,0,0），便于覆盖
  • @layer 引入层级：layer 内样式优先级低于非 layer 样式（无论权重）；多个 layer 按声明顺序后者优先
  • :not(.a, .b) Level 4 语法：多参数取非
  • 伪元素 ::slotted / ::part 穿透 Shadow DOM
现代选择器（Level 3/4）★ 新
:has() 关系伪类
"父选择器"正式到来。div:has(> img) 选含直接子图片的 div。label:has(+ input:invalid) 选后面有非法输入的 label。Chrome 105+、Safari 15.4+、Firefox 121+。
:is() / :where()
简化复杂选择器：:is(h1,h2,h3) a 代替三行规则。:where() 同功能但权重为 0，适合写基础样式库（方便用户覆盖）。
逻辑伪类
  • :nth-child(2n+1 of .item) Level 4：按类型计数
  • :user-invalid — 用户交互后才显示错误（比 :invalid 更友好）
  • :focus-visible — 键盘焦点环（鼠标点击不显示）
  • :focus-within — 子元素 focus 时激活
  • :any-link — 匹配所有链接（:link + :visited）
CSS 嵌套（原生）★ 新
无需预处理器：
.card { color: red; /* 直接嵌套（需 & 或选择器）*/ &:hover { color: blue; } .title { font-size: 1.2em; } @media (width > 600px) { ... } }
@scope ★ 新
限定选择器作用域：@scope (.card) { .title { color:red } }。可设上下界：@scope (.card) to (.slot) 精确控制范围，替代 BEM 命名方案。Chrome 118+。
::part() & ::slotted()
Web Components 样式穿透：::part(button) 选择 Shadow DOM 中有 part="button" 的元素。::slotted(p) 选择插槽中分发的 p 元素。exportparts 传递 part 到外层。
@layer 层叠层 ★ 新高频
💡
@layer 解决的核心问题：管理样式优先级不再依赖权重竞争。Layer 内样式统一低于非 Layer 样式；多 Layer 按声明顺序，后者优先。
@layer reset, base, components, utilities;（先声明顺序，后填充内容）
!important 在 layer 中的行为与普通样式相反：越早的 layer，!important 优先级越高。
继承与初始值
关键字含义典型用途
inherit强制继承父元素值border 不继承，强制 border:inherit
initial重置为规范初始值（非浏览器默认）color initial = black（非 user-agent 蓝）
unset继承属性=inherit，非继承=initial重置表单样式
revert回退到 user-agent 默认样式在组件库中安全重置
revert-layer回退到上一个 @layer 的值@layer 体系内样式撤销
all: unset / revert重置所有属性彻底重置按钮/输入框样式

CSS 变量 & 函数
Variables & Functions
CSS 自定义属性（变量）★ 高频
核心特性
  • 作用域：继承机制（可在 :root 全局定义，也可局部覆盖）
  • var(--color, fallback) 第二参数为回退值
  • 可存任何值（包括空格、无效值）；无效值不会报错，使用 initial
  • JS 读取：getComputedStyle(el).getPropertyValue('--x')
  • JS 设置：el.style.setProperty('--x', val)
  • 不参与计算（calc 需在 var 外包计算）
  • @property 注册类型：支持动画、语法约束、初始值
@property 注册自定义属性 ★ 新
@property --hue { syntax: '<angle>'; inherits: false; initial-value: 0deg; } /* 现在 --hue 可以被 transition 动画！*/ .el { transition: --hue 1s; }
数学函数
calc() / min() / max() / clamp()
  • calc(100% - 2rem) 混合单位计算
  • min(50%, 300px) 取小（响应式上限）
  • max(200px, 30%) 取大（响应式下限）
  • clamp(min, val, max) = max(min, min(val, max))。经典字体：clamp(1rem, 2.5vw, 2rem)
三角 & 高级数学 ★ 新
  • sin() cos() tan() asin() acos() atan() atan2()
  • pow() sqrt() log() exp()
  • abs() round() mod() rem() sign()
  • 用途：纯 CSS 时钟、轮播角度计算、无JS动画
  • Chrome 111+、Firefox 108+
颜色函数 ★ 新
  • oklch(L C H) 感知均匀色彩空间（推荐）
  • color-mix(in oklch, red 50%, blue) 颜色混合
  • color(display-p3 1 0 0) 广色域
  • light-dark(light-val, dark-val) 自动适配主题
  • oklch 比 hsl 更一致（不同色相亮度感知相同）
变换函数
  • translate(x,y) / translateX/Y/Z
  • scale(x,y) / rotate(deg)
  • skew(x,y) / matrix(a,b,c,d,e,f)
  • perspective() 在 transform 内使用影响单元素
  • 新：translate: x y（独立属性，可独立 transition）
  • 新：rotate: 45deg / scale: 1.2 独立属性
图形函数（clip-path / shape）
  • clip-path: polygon(0 0, 100% 0, 100% 80%, 0 100%)
  • clip-path: circle(50%) / ellipse() / inset()
  • shape-outside：文字环绕自定义形状
  • 新：CSS Shapes Level 2 shape() 函数（路径命令）
滤镜函数
  • filter: blur() brightness() contrast() saturate() grayscale() hue-rotate() drop-shadow()
  • backdrop-filter 模糊背后内容（毛玻璃效果）
  • filter:drop-shadow vs box-shadow：前者跟随 PNG 透明轮廓

动画 & 过渡
Animation
transition vs animation 对比 ★ 高频
transition（过渡）
  • 需要状态触发（:hover、class切换、JS）
  • 从 A 到 B 的单次过渡，不循环
  • transition: property duration timing-function delay
  • 多属性用逗号分隔；all 性能差（避免）
  • transition-behavior: allow-discrete（切换 display 的过渡，2023+）
animation（关键帧动画）
  • 独立运行，可循环、延迟、暂停
  • animation: name duration timing iteration direction fill-mode play-state
  • animation-fill-mode: forwards 保持最终状态
  • animation-iteration-count: infinite
  • animation-play-state: paused JS 控制暂停
  • Web Animations API（JS）可获取 Animation 对象控制
缓动函数深解
timing-function 类型
  • linear — 匀速
  • ease / ease-in / ease-out / ease-in-out — 预设贝塞尔
  • cubic-bezier(x1,y1,x2,y2) — 自定义贝塞尔，y 可超出 0-1（弹性效果）
  • steps(n, start|end) — 阶梯动画（精灵图翻页）
  • step-start = steps(1, start)；step-end = steps(1, end)
  • 新：linear(0, 0.5 25%, 1) 多点线性（模拟 spring 弹簧）
可动画属性 & GPU 加速
  • 只走 Composite（最快）：transform、opacity
  • 仅 Paint：color、background-color、box-shadow
  • 触发 Layout+Paint+Composite（最慢）：width、height、top/left、margin、padding
  • will-change: transform 提示浏览器提升合成层（勿滥用）
  • 新可动画：display（需 allow-discrete）、height: auto（calc-size）
Scroll-driven Animation ★ 新高频
滚动驱动动画
无需 JS 监听 scroll 事件，纯 CSS 实现滚动联动：
@keyframes fadeIn { from { opacity:0 } } .el { animation: fadeIn linear; animation-timeline: scroll(); /* 或 view() — 元素进入视口 */ animation-timeline: view(); animation-range: entry 0% entry 100%; }
Chrome 115+，Firefox 2024+
View Transition（CSS 部分）★ 新
  • view-transition-name: hero 标记参与过渡的元素
  • ::view-transition-old(hero) 旧状态截图伪元素
  • ::view-transition-new(hero) 新状态伪元素
  • 默认淡入淡出；可自定义 @keyframes 覆盖
  • MPA 跨页面过渡需 @view-transition { navigation: auto }
@starting-style & allow-discrete ★ 新
💡
2023 新特性组合：元素进入/退出动画。
@starting-style 定义元素首次渲染时的起始样式（解决 display:none → block 无法 transition 问题）。配合 transition-behavior: allow-discrete 实现 display 属性的过渡。Chrome 117+、Safari 17.5+。

现代新特性（2022—2025）
Modern CSS
布局新能力
Masonry Layout ★ 草案
瀑布流布局。grid-template-rows: masonry（Chrome 实验/Safari TP）。无需 JS 计算高度，浏览器自动填充最短列。规范争议：Chrome 推 grid 扩展，Firefox 提独立属性。
anchor positioning ★ 新
将元素定位相对于任意"锚点"元素：anchor-name: --btn + position-anchor: --btn + top: anchor(bottom)。彻底解决 tooltip/popover 相对定位问题。Chrome 125+。
calc-size() ★ 新
支持 height: auto 参与计算：height: calc-size(auto, size + 20px)。可对 auto/min-content/max-content 进行动画过渡，解决"展开折叠"动画的历史难题。Chrome 129+。
样式与主题
color-scheme & prefers
  • color-scheme: light dark 让浏览器 UA 样式也跟随主题
  • :root { color-scheme: light dark } 全局启用
  • light-dark(#fff, #000) 单行实现主题色
  • 表单控件、滚动条也自动适配暗色
CSS 嵌套（原生）+ @scope
  • 原生嵌套全主流浏览器支持（2023+）
  • 不再依赖 Sass/Less 进行嵌套
  • @scope 提供真正的局部样式作用域
  • 两者组合 ≈ CSS Modules 效果，无需构建工具
新文本特性
  • text-wrap: balance — 多行标题自动均衡换行
  • text-wrap: pretty — 优化段落最后行孤字
  • overflow-wrap: anywhere — 任意字符换行
  • hanging-punctuation — 标点悬挂（Safari）
  • initial-letter — 首字下沉
逻辑属性 ★ 高频
  • 物理 → 逻辑：margin-left → margin-inline-start
  • padding-block（上下）/ padding-inline（左右）
  • inset-inline-start 替代 left
  • block-size = height；inline-size = width
  • 支持 RTL 语言自动翻转，国际化必备
font 新特性
  • font-display: swap|block|fallback|optional
  • font-variation-settings — 可变字体轴控制
  • font-palette / @font-palette-values — 彩色字体主题
  • size-adjust / ascent-override / descent-override — 回退字体指标匹配（减少 CLS）

  • overflow: clip — 裁剪但不创建滚动容器（比 hidden 更安全）

  • scroll-behavior: smooth — 原生平滑滚动

visibility:hiddenvsopacity:0 visibility:hidden 子元素可单独 visible；opacity:0 整体透明且子元素无法覆盖，但仍可捕获事件（需加 pointer-events:none）
display:nonevsvisibility:hidden display:none 脱离渲染树（不占空间，不可捕获事件）；visibility:hidden 在渲染树中（占空间，触发 Paint）
emvsrem em 相对于当前元素 font-size（复合继承，嵌套时容易失控）；rem 始终相对 :root font-size（可预测）
padding %vsmargin % padding/margin 的百分比值均相对于包含块的宽度（不是高度），这是等比例正方形技巧的原理（现已被 aspect-ratio 取代）
position:absolutevsposition:fixed absolute 相对最近定位祖先；fixed 相对视口——但有 transform/filter 祖先时 fixed 变为相对该祖先
overflow:hiddenvsoverflow:clip hidden 创建 BFC（影响布局）、创建滚动容器；clip 仅裁剪，不创建 BFC 和滚动容器，不影响布局流（更安全）
:nth-child(n)vs:nth-of-type(n) nth-child 在所有兄弟中按位置计数；nth-of-type 只在同标签兄弟中计数。混用时极易出错
transition: allvs指定属性 all 会尝试过渡所有可动画属性，性能差且无法控制；应明确指定属性列表
flex-basis:autovsflex-basis:0 auto 以内容/width 为初始尺寸；0 忽略内容尺寸，按 flex-grow 纯比例分配（flex:1 的行为）
z-index 失效vs有效 z-index 只在 position≠static 或 flex/grid 子项上生效；且层叠上下文内的 z-index 不能与上下文外比较
性能优化 & 避坑
CSS 性能最佳实践
  • 动画只用 transform + opacity（Compositor 线程，不阻塞主线程）
  • 避免 @import（串行加载），改用 <link> 或打包
  • 大量节点用 content-visibility:auto 跳过视口外渲染
  • contain:strict 隔离昂贵组件的布局/绘制
  • 关键 CSS 内联，非关键异步加载（Critical CSS）
  • 用 will-change 提前创建合成层，但避免滥用（显存压力）
选择器性能
  • CSS 选择器从右往左匹配（关键选择器优化最重要）
  • 避免通配符 * { ... } 在深层（匹配所有元素）
  • 避免超长选择器链：.a .b .c .d .e
  • :has() 在大型文档中可能有性能影响（浏览器持续优化）
  • BEM 命名的性能优势：扁平单类选择器，匹配最快
面试高频原理题速查
问题核心答案
清除浮动几种方式 ① display:flow-root（最推荐，无副作用）② clearfix 伪元素 ③ overflow:hidden/auto（创建BFC，有副作用）④ 额外 clear:both 元素（不推荐）
垂直居中方案 ① display:flex; align-items:center ② display:grid; place-items:center ③ position:absolute; top:50%; transform:translateY(-50%) ④ margin:auto（absolute + inset:0）⑤ line-height（单行文字）
CSS 加载阻塞渲染吗 CSS 是渲染阻塞资源（必须下载+解析完才构建渲染树）。JS 执行会等 CSSOM 就绪（CSS 间接阻塞 JS）。link 并行加载但阻塞渲染；@import 串行，加剧阻塞。
什么触发 GPU 合成层 transform/opacity 动画（有 will-change 或浏览器判断）、will-change: transform、video/canvas/iframe、有 CSS filter、position:fixed（部分情况）
CSS 变量 vs Sass 变量 CSS 变量：运行时（JS可改、响应媒体查询、可继承作用域）。Sass 变量：编译时（静态替换，无运行时能力）。两者可互补使用。
如何实现主题切换 ① CSS 变量 + [data-theme="dark"] 切换根选择器 ② prefers-color-scheme 自动跟随系统 ③ color-scheme 属性让 UA 样式也适配 ④ light-dark() 函数单行声明
1px 问题 设备像素比（DPR）>1 时，1px CSS = 多个物理像素。方案：transform:scaleY(0.5)、border-image、box-shadow: 0 0.5px、@media (-webkit-min-device-pixel-ratio:2) + SVG/图片边框
CSS Houdini 是什么 浏览器暴露渲染引擎底层 API 给开发者：Paint Worklet（自定义背景/边框绘制）、Layout Worklet（自定义布局算法）、Animation Worklet（自定义动画）、@property（类型化变量）。目前 @property 最成熟。
